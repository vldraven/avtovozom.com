# Инструкция по заполнению конфига калькулятора растаможки

Калькулятор использует библиотеку **tks-api-official** и YAML с корневым ключом `tariffs`. Рабочий текст хранится в БД и редактируется в админке: **Профиль → Настройки калькулятора растаможки** (`/staff/admin-customs-calculator`). Шаблон по умолчанию задаётся в коде (`DEFAULT_TKS_CONFIG_YAML` в `backend/app/customs_calc.py`); копия с комментариями для справки — `backend/app/customs_tariffs.example.yaml`.

## Общие правила

- Строки, начинающиеся с `#`, — **комментарии**: сохраняются в базе и **не влияют** на расчёт.
- После сохранения API **не пересобирает** YAML через парсер обратно в текст — комментарии не пропадают.
- Любое изменение должно оставаться **валидным YAML** и проходить проверку при сохранении (загрузка в `CustomsCalculator`).

## Поля `tariffs`

| Поле | Смысл |
|------|--------|
| `base_clearance_fee` | Сумма таможенного сбора за оформление (₽). В модели **одно число**; по закону сбор зависит от таможенной стоимости (ступени). |
| `base_util_fee` | Слагаемое «Util Fee» в режиме ETC; в CTP участвует как `base_util_fee × ctp_util_coeff_base`. Не тождественно утильсбору по ПП РФ. |
| `etc_util_coeff_base` | В текущей версии **tks-api-official** в `calculate_etc` **не используется**; оставлено для совместимости с апстримом. |
| `ctp_util_coeff_base` | Только режим **CTP**: множитель к `base_util_fee`. |
| `excise_rates` | Режим **CTP**: акциз = мощность (л.с.) × ставка (₽/л.с.). В НК РФ ставки **ступенчатые по мощности**; здесь одно число на тип двигателя — упрощение. |
| `recycling_factors` | В библиотеке: **20 000 ₽ × коэффициент** (база зашита в коде). Не заменяет полный расчёт утильсбора по ПП РФ № 1291 / № 81. |
| `age_groups.overrides` | Для каждого возраста (`new`, `1-3`, `3-5`, `5-7`, `over_7`) и типа двигателя (`gasoline`, `diesel`, `electric`, `hybrid`). |
| `rate_per_cc` | Условные **«евро за см³»** для формулы ETC: пошлина ≈ `rate_per_cc × объём_см³ × курс(1 EUR → RUB)`. Ориентир сверки — таблицы ЕТТ для **физлиц**; в дефолте подобрано под эталон **~2000 см³** (см. комментарии в example). |
| `min_duty` | В коде калькулятора **не читается**; можно оставить для заметок администратору. |
| `individual_etc_util_fee` | Только **ETC** при ввозе **физлицом**: сумма в ₽ для строки «Util Fee» вместо полного `base_util_fee` из библиотеки tks (по умолчанию в проекте **0**). |
| `individual_recycling_rub` | Необязательно: зафиксировать утилизационный сбор для **ETC физлица** (₽). Если не задано — по-прежнему **20 000 × коэффициент** из `recycling_factors`. |

На бэкенде дополнительно: **таможенный сбор** в ответах подставляется **по ступеням** от таможенной стоимости в ₽ (как в актуальных таблицах), а курсы валют для расчёта берутся с **ЦБ РФ** (тот же XML, что и для CNY на сайте), а не из `currency_converter_free` в tks-api.

В ответе API поле **`etc_context`** дублирует выбранный возраст и фактическую **`rate_per_cc`** из YAML. Если при смене возраста **итог ETC не меняется**, чаще всего в конфиге для этих возрастных групп задана **одна и та же** `rate_per_cc` (в дефолтном примере совпадают «Новый», «1–3» и «3–5» — все **2,7**). Чтобы суммы различались, задайте в админке **разные** `rate_per_cc` по группам или сравнивайте с группами «5–7» / «Старше 7», где обычно другая ставка.

Режимы ответа API:

- **ETC** (`etc`) — ближе к оценке для физлица (ставка за см³ + сборы).
- **CTP** (`ctp`) — упрощённая схема для **юрлица** (проценты, НДС, акциз по мощности); не полный юридический расчёт импортёра.

Подробнее о формулах см. исходник: [`tks_api_official/calc.py`](https://github.com/markolofsen/tks-api/blob/main/tks_api_official/calc.py) в репозитории tks-api.

## Как обновлять ставки

1. Сверяйте `rate_per_cc` с актуальными **едиными ставками для физлиц** (ЕТТ / публикации ЕЭК, справочные таблицы). Учтите: **одна цифра на возраст** не покрывает все диапазоны объёма двигателя и не описывает режим **авто до трёх лет** (там % и минимумы в EUR по стоимости).
2. Ставки акциза для CTP сверяйте с **НК РФ** (ступени по л.с.).
3. После правок сохраните конфиг в админке; при ошибке валидации прочитайте текст ошибки от API.
