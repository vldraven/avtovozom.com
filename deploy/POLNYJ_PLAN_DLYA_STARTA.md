# Полный пошаговый план (для нетехнического пользователя)

Вы уже **создали пустой репозиторий на GitHub**. Ниже — что делать дальше, по шагам. Делайте по порядку; не пропускайте шаги внутри блока.

**Что понадобится:** Mac, доступ в интернет, логин GitHub, пароль или SSH к серверу (VPS), доступ в REG.RU к домену **avtovozom.com**.

**Важно:** замените в командах **ваш_логин** и **имя_репозитория** на то, что у вас на GitHub (например логин `ivan`, репозиторий `avtovozom`).

---

## Этап 1. Положить код с компьютера на GitHub

### Шаг 1.1. Открыть программу «Терминал» на Mac

- Через Spotlight (Cmd+Пробел) введите `Терминал` / `Terminal` и откройте приложение.
- Чёрное или белое окно с текстом — это нормально.

### Шаг 1.2. Перейти в папку проекта

Скопируйте команду целиком, вставьте в Терминал (Cmd+V), нажмите **Enter**:

```bash
cd /Users/vladislavgusynin/Documents/avtovozom.com
```

Если папка проекта у вас в другом месте — замените путь на свой.

### Шаг 1.3. Проверить, что Git видит проект

```bash
git status
```

Должно написать что-то вроде `On branch main` и список файлов или `nothing to commit`. Если ошибка `not a git repository` — напишите в поддержку или разработчику: в проекте должен быть каталог `.git`.

### Шаг 1.4. Подключить GitHub к этой папке (remote)

**Один раз** выполните (подставьте **свой** логин GitHub и **точное** имя репозитория):

```bash
git remote add origin https://github.com/ваш_логин/имя_репозитория.git
```

Если появится ошибка `remote origin already exists` — значит remote уже добавляли. Тогда проверьте адрес:

```bash
git remote -v
```

Если там неправильный URL — удалите и добавьте снова:

```bash
git remote remove origin
git remote add origin https://github.com/ваш_логин/имя_репозитория.git
```

### Шаг 1.5. Выложить код на GitHub (push)

Убедитесь, что ветка называется `main`:

```bash
git branch -M main
```

Отправьте код:

```bash
git push -u origin main
```

**Что может произойти:**

1. **Спросит логин и пароль** — логин: ваш GitHub username. **Пароль НЕ от GitHub:** нужен **Personal Access Token** (см. шаг 1.6).
2. **Ошибка про `rejected` / `non-fast-forward`** — на GitHub при создании репозитория вы поставили галочку «README». Тогда сначала:
   ```bash
   git pull origin main --rebase
   git push -u origin main
   ```
   Если не получится — сохраните текст ошибки и обратитесь за помощью.
3. **Успех** — в браузере откройте страницу репозитория на GitHub: должны появиться папки `backend`, `web`, файлы.

### Шаг 1.6. Как сделать пароль для Git (Personal Access Token)

На сайте GitHub (в браузере):

1. В правом верхнем углу: аватар → **Settings**.
2. Слева внизу: **Developer settings** → **Personal access tokens** → **Tokens (classic)**.
3. **Generate new token (classic)**.
4. Название любое, срок — по желанию, отметьте права **`repo`** (полный доступ к репозиториям).
5. **Generate token** — **скопируйте** длинную строку сразу (потом её не покажут).
6. В Терминале при `git push`, когда просят **Password**, вставьте **этот токен** (не обычный пароль от сайта).

Альтернатива: установить **GitHub Desktop** и через него опубликовать репозиторий — тогда меньше работы с токеном в Терминале.

## Этап 2. Отдельный ключ: чтобы GitHub сам «заходил» на сервер и обновлял сайт

Это **второй** ключ, не путайте с логином на GitHub. Он только для автоматического деплоя.

### Шаг 2.1. Создать ключ на Mac

В Терминале:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/avtovozom_deploy -N "" -C "github-deploy"
```

Если спросит перезапись файла — если вы уже создавали этот ключ раньше и уверены, можно `y`, иначе **не перезаписывайте**, используйте уже существующий.

### Шаг 2.2. Показать «публичную» часть ключа (одна строка)

```bash
cat ~/.ssh/avtovozom_deploy.pub
```

Скопируйте **всю одну длинную строку** (начинается с `ssh-ed25519`).

### Шаг 2.3. Вставить эту строку на сервер

1. Подключитесь к серверу (как вы уже умеете):
   ```bash
   ssh root@91.196.33.68
   ```
   (Если у вас другой пользователь или IP — подставьте свои.)

2. На сервере выполните:
   ```bash
   mkdir -p ~/.ssh
   chmod 700 ~/.ssh
   nano ~/.ssh/authorized_keys
   ```

3. Откроется редактор: стрелками вниз **в самый конец файла**, вставьте **одну строку** из буфера (публичный ключ), с новой строки.
4. Сохранить: **Ctrl+O**, Enter, выйти: **Ctrl+X**.

5. Права:
   ```bash
   chmod 600 ~/.ssh/authorized_keys
   exit
   ```

### Шаг 2.4. Проверка с Mac

```bash
ssh -i ~/.ssh/avtovozom_deploy root@91.196.33.68
```

Должно войти **без пароля**. Напишите `exit` чтобы выйти.

## Этап 3. Секреты в GitHub (без них деплой не запустится)

1. Откройте в браузере **ваш репозиторий** на GitHub.
2. Вкладка **Settings** (настройки репозитория, не всего аккаунта).
3. Слева: **Secrets and variables** → **Actions**.
4. **New repository secret** — создайте **четыре** секрета **точно с такими именами**:

| Имя секрета (копировать как есть) | Что вписать в Value |
|-----------------------------------|---------------------|
| `DEPLOY_HOST` | IP сервера, например `91.196.33.68` |
| `DEPLOY_USER` | `root` (или ваш пользователь Linux на сервере) |
| `DEPLOY_PATH` | `/opt/avtovozom` — **только если проект лежит именно там**. Если у вас другой путь — напишите его **без** слэша в конце. |
| `DEPLOY_SSH_KEY` | **Приватный** ключ целиком (см. ниже) |

**Как заполнить `DEPLOY_SSH_KEY`:**

На Mac в Терминале:

```bash
cat ~/.ssh/avtovozom_deploy
```

Скопируйте **всё** от строки `-----BEGIN OPENSSH PRIVATE KEY-----` до `-----END OPENSSH PRIVATE KEY-----` **включительно**. Вставьте в поле Value секрета на GitHub (одним куском).

## Этап 4. Подготовка сервера (один раз)

Подключитесь по SSH к серверу и выполните по очереди.

### Шаг 4.1. Папка проекта и медиа

```bash
mkdir -p /opt/avtovozom/media
```

### Шаг 4.2. Файл `.env` на сервере

Если вы **ещё не создавали** `.env` на сервере:

```bash
cd /opt/avtovozom
nano .env
```

Вставьте содержимое по шаблону из файла в проекте **`deploy/env.production.example`** (он есть на GitHub после push) или скопируйте с Mac:

На **Mac** (новое окно Терминала):

```bash
scp /Users/vladislavgusynin/Documents/avtovozom.com/deploy/env.production.example root@91.196.33.68:/opt/avtovozom/.env.example_on_server
```

На **сервере** откройте и перепишите в `.env` с реальными паролями и URL.

Минимум в `.env`: `POSTGRES_PASSWORD`, `NEXT_PUBLIC_API_URL=https://api.avtovozom.com`, `CORS_ORIGINS`, `PUBLIC_WEB_ORIGIN`. SMTP можно позже.

### Шаг 4.3. Docker на сервере

Если Docker ещё не ставили — используйте официальную инструкцию для Ubuntu/Debian или раздел в **`deploy/PRODUCTION.md`**. Проверка:

```bash
docker --version
docker compose version
```

## Этап 5. Первый автоматический деплой

1. На Mac в папке проекта сделайте маленькое изменение (например пробел в README) или просто убедитесь, что всё закоммичено:
   ```bash
   cd /Users/vladislavgusynin/Documents/avtovozom.com
   git status
   ```
   Если есть изменения:
   ```bash
   git add -A
   git commit -m "Проверка деплоя"
   git push origin main
   ```

2. На GitHub откройте вкладку **Actions**.
3. Слева выберите workflow **Deploy production**.
4. Должен появиться жёлтый/зелёный запуск. Откройте его — смотрите логи. **Зелёная галочка** = успех.

Если **красный крест** — откройте шаг **Rsync and docker compose**, прочитайте красный текст ошибки (часто неверный путь, нет Docker, нет прав, неверный ключ).

## Этап 6. Домен avtovozom.com

### Шаг 6.1. DNS в REG.RU

В разделе управления DNS **зоны** (не смена NS-серверов на произвольные):

- **A** для **`@`** (корень) → IP вашего VPS  
- **A** для **`www`** → тот же IP  
- **A** для **`api`** → тот же IP  

Сохранить. Подождать от 5 минут до пары часов.

### Шаг 6.2. Caddy (HTTPS) на сервере

Кратко (подробнее в **`deploy/PRODUCTION.md`** и **`deploy/Caddyfile`**):

1. Установить Caddy на сервер.
2. Скопировать пример конфига из репозитория `deploy/Caddyfile` в `/etc/caddy/Caddyfile`.
3. В начале файла указать **реальный email** в строке `email ...` для сертификатов Let’s Encrypt.
4. Выполнить:
   ```bash
   sudo caddy validate --config /etc/caddy/Caddyfile
   sudo systemctl reload caddy
   ```

### Шаг 6.3. Firewall

Должны быть открыты порты **80** и **443** (и **22** для SSH). Порты **3000** и **8000** наружу открывать не нужно — сайт идёт через Caddy.

## Как жить дальше (обновления сайта)

1. Меняете файлы на Mac (или в Cursor).
2. В Терминале:
   ```bash
   cd /Users/vladislavgusynin/Documents/avtovozom.com
   git add -A
   git commit -m "Кратко что изменили"
   git push origin main
   ```
3. GitHub Actions сам обновит сервер (через 1–3 минуты обычно).

Файл **`.env`** на сервере и папка **`media`** при деплое **не затираются**.

## Если что-то пошло не так

- **Не пушится на GitHub** — токен, правильность URL remote, интернет.
- **Actions падает** — логи Actions; проверьте четыре секрета; проверьте `DEPLOY_PATH` (существует ли папка на сервере).
- **Сайт не открывается** — DNS, Caddy, `docker compose ps` на сервере, логи контейнеров.

Сохраните **текст ошибки** из Терминала или GitHub — по нему проще помочь точечно.
